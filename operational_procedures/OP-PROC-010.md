# School District Onboarding Procedure (OP-PROC-010)

## 1. Purpose

This procedure provides step-by-step instructions for onboarding new school and district customers, ensuring all required agreements are in place and data protections are established before receiving student data.

## 2. Scope

This procedure applies to Sales, Legal, Implementation, and Customer Success teams involved in customer onboarding.

## 3. Procedure

### 3.1 Pre-Contract Phase

#### 3.1.1 Customer Qualification

**When:** Initial engagement with prospective school customer.

**Steps:**

1. **Gather customer information**
   - School or district name
   - State(s) of operation
   - K-12 or higher education
   - Estimated number of students
   - Products/services of interest

2. **Identify compliance requirements**
   - Applicable state laws (see ANNEX-003)
   - FERPA requirements
   - COPPA requirements (if K-12)
   - Any special requirements (e.g., state DPA)

3. **Document in CRM**
   - Customer profile
   - Compliance requirements
   - Sales stage

#### 3.1.2 Security Questionnaire Response

**When:** Customer sends security/privacy questionnaire.

**Steps:**

1. **Identify questionnaire type**
   - HECVAT (higher ed)
   - State template (CA, NY, etc.)
   - Custom questionnaire

2. **Route to appropriate teams**
   - Security questions → Security team
   - Privacy questions → Privacy/Legal team
   - Product questions → Product team

3. **Complete questionnaire**
   - Use standard responses where applicable
   - Customize for customer context

4. **Review before submission**
   - Accuracy check
   - Consistency check

5. **Submit to customer**
   - Track submission
   - Follow up as needed

#### 3.1.3 Compliance Documentation Package

**When:** Customer requests compliance documentation.

**Steps:**

1. **Prepare package**
   - Privacy policy
   - Standard DPA or customer's DPA (signed)
   - SOC 2 report summary (full report under NDA)
   - VPAT
   - Security overview
   - Insurance certificates (upon request)

2. **Customize as needed**
   - State-specific documentation
   - Customer-specific responses

3. **Deliver to customer**
   - Secure delivery method
   - Track delivery

### 3.2 Contracting Phase

#### 3.2.1 DPA Preparation

**When:** Ready to contract with customer.

**Steps:**

1. **Determine DPA approach**
   - Customer provides DPA → Section 3.2.2
   - State standard DPA available → Section 3.2.3
   - Use **[Company Name]** standard DPA → Section 3.2.4

2. **Gather required information**
   - School/district legal name
   - Authorized signatory
   - Services to be provided
   - Data elements to be processed
   - State-specific requirements

#### 3.2.2 Customer-Provided DPA

**When:** Customer provides their own DPA.

**Steps:**

1. **Receive customer DPA**

2. **Route for legal review**
   - Legal reviews terms
   - Identifies any concerns
   - Proposes redlines if needed

3. **Negotiate terms**
   - Communicate redlines to customer
   - Work toward acceptable terms
   - Escalate if impasse

4. **Finalize DPA**
   - Both parties agree to final terms
   - Proceed to execution

#### 3.2.3 State Standard DPA

**When:** Customer state has standardized DPA (SDPC, state template).

**Steps:**

1. **Identify applicable standard**
   - SDPC National DPA
   - California NDPA
   - State-specific template

2. **Complete standard DPA**
   - Fill required fields
   - Add exhibits/attachments

3. **Execute per state process**
   - Use state portal if available
   - Or execute directly with district

4. **Document in contract system**

#### 3.2.4 Company Standard DPA

**When:** Using **[Company Name]** standard DPA.

**Steps:**

1. **Prepare DPA**
   - Use template (ANNEX-005)
   - Complete variable fields
   - Add state-specific addenda if needed

2. **Add required exhibits**
   - Service description
   - Data elements
   - Subcontractor list
   - Security measures

3. **Send to customer**
   - Request review and signature

4. **Address customer questions**
   - Provide clarifications
   - Minor modifications if needed

#### 3.2.5 DPA Execution

**When:** DPA terms agreed.

**Steps:**

1. **Obtain customer signature**
   - Verify signatory is authorized
   - Confirm signature date

2. **Countersign for **[Company Name]****
   - Authorized signatory signs
   - Date countersignature

3. **Distribute fully executed DPA**
   - Send copy to customer
   - Retain copy in contract system

4. **File DPA**
   - Upload to contract management
   - Tag with customer, dates, state
   - Set renewal reminder

### 3.3 Implementation Phase

#### 3.3.1 Implementation Kickoff

**When:** DPA executed, ready to implement.

**Steps:**

1. **Schedule kickoff meeting**
   - Customer stakeholders
   - **[Company Name]** implementation team

2. **Confirm implementation scope**
   - Products/services
   - Schools/sites
   - User populations
   - Timeline

3. **Identify contacts**
   - Customer technical contact
   - Customer admin contact
   - Data privacy contact
   - **[Company Name]** contacts

4. **Document kickoff**
   - Meeting notes
   - Action items
   - Timeline

#### 3.3.2 Data Inventory

**When:** Preparing to receive student data.

**Steps:**

1. **Complete data inventory** (ANNEX-004)
   - Data elements to be received
   - Data sources (SIS, roster, manual)
   - Purpose for each element
   - Retention requirements

2. **Review with customer**
   - Confirm data elements match DPA
   - Address any discrepancies

3. **File data inventory**
   - Store in customer record
   - Reference for future requests

#### 3.3.3 Technical Setup

**When:** Configuring customer environment.

**Steps:**

1. **Create customer environment**
   - Provision tenant/account
   - Configure per customer requirements

2. **Set up integrations** (if applicable)
   - SSO integration
   - Rostering (Clever, ClassLink, OneRoster)
   - SIS integration

3. **Configure access controls**
   - Admin accounts
   - Role-based access per customer
   - Data segregation verified

4. **Configure privacy settings**
   - Per customer preferences
   - Per state requirements

5. **Document configuration**
   - Settings and integrations
   - Test results

#### 3.3.4 Data Import

**When:** Ready to receive student data.

**Steps:**

1. **Confirm data transfer method**
   - Automated rostering (preferred)
   - Secure file transfer
   - Manual entry

2. **Receive data**
   - Secure transmission
   - Validate data format
   - Check for errors

3. **Import data**
   - Load into system
   - Verify import success
   - Address any errors

4. **Confirm with customer**
   - Data loaded correctly
   - Ready for user access

#### 3.3.5 Training

**When:** System configured, data loaded.

**Steps:**

1. **Schedule training**
   - Administrators
   - Teachers/staff
   - Students (if applicable)

2. **Deliver training**
   - Product features
   - Privacy settings
   - Reporting
   - Support resources

3. **Provide materials**
   - Training documentation
   - Quick reference guides
   - Support contact info

4. **Document training**
   - Attendance
   - Topics covered
   - Follow-up items

### 3.4 Go-Live

#### 3.4.1 Go-Live Checklist

**When:** Ready to launch.

**Steps:**

1. **Verify readiness**
   - [ ] DPA executed
   - [ ] Technical setup complete
   - [ ] Data imported
   - [ ] Training delivered
   - [ ] Customer accepts readiness

2. **Enable production access**
   - Activate user accounts
   - Open to users

3. **Monitor initial usage**
   - Watch for issues
   - Respond to questions

4. **Go-live confirmation**
   - Customer confirms successful launch
   - Document go-live date

#### 3.4.2 Handoff to Customer Success

**When:** Go-live complete.

**Steps:**

1. **Prepare handoff documentation**
   - Customer overview
   - Configuration details
   - Known issues/notes
   - Key contacts

2. **Conduct handoff meeting**
   - Implementation team
   - Customer Success team
   - Review customer details

3. **Transfer relationship**
   - Customer Success primary contact
   - Implementation available for escalation

4. **Document handoff**
   - Handoff date
   - Customer Success owner

### 3.5 Post-Onboarding

#### 3.5.1 Complete Customer Record

**When:** Onboarding complete.

**Steps:**

1. **Verify documentation complete**
   - [ ] Executed DPA
   - [ ] Data inventory
   - [ ] Configuration documentation
   - [ ] Training records
   - [ ] Go-live confirmation

2. **Update CRM**
   - Status: Active customer
   - Key dates recorded
   - Contacts current

3. **Set renewal reminder**
   - DPA expiration
   - Contract renewal date

## 4. Escalation

Escalate to management when:
- DPA negotiation at impasse
- Customer data breach concern
- Implementation significantly delayed
- Customer complaint

## 5. Records

| Record | Retention | Location |
|--------|-----------|----------|

| Security questionnaires | 3 years | Customer record |
| Data inventories | Contract + 3 years | Customer record |
| Training records | 3 years | Customer record |
| Implementation documentation | Contract + 3 years | Customer record |
