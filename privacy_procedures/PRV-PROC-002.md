# COPPA Compliance Procedure (PRV-PROC-002)

## 1. Purpose

This procedure provides step-by-step instructions for ensuring COPPA compliance when collecting personal information from children under 13, including consent management, notice requirements, and parental rights.

## 2. Scope

This procedure applies to all workforce members who design, develop, or operate products used by children under 13.

## 3. Procedure

### 3.1 COPPA Applicability

#### 3.1.1 Determining COPPA Applicability

**When:** Launching new product/feature or entering new market.

**Steps:**

1. **Evaluate "directed to children"**
   - Subject matter appeals to children?
   - Visual design targets children?
   - Animated characters or child activities?
   - Marketing targets children or schools?
   - Used primarily in K-12 settings?

2. **Evaluate "actual knowledge"**
   - Will we know users are under 13?
   - Age collected during registration?
   - School/grade level indicates age?

3. **Determine applicability**
   - If directed to children OR actual knowledge → COPPA applies
   - K-12 products: presume COPPA applies

4. **Document determination**
   - Record analysis
   - If COPPA applies → implement requirements

### 3.2 School Consent Model

#### 3.2.1 Obtaining School Consent

**When:** Providing services to K-12 schools where children under 13 will use the product.

**Steps:**

1. **Verify school consent criteria**
   - Service used solely for educational purpose
   - School benefits for educational purposes
   - No commercial use unrelated to education

2. **Provide notice to school**
   - Types of personal information collected
   - How information will be used
   - Disclosure practices
   - Privacy policy URL
   - Contact information

3. **Obtain written consent**
   - Executed DPA constitutes consent
   - OR explicit written authorization
   - Document consent received

4. **Verify school authority**
   - School representative is authorized
   - School has determined parents are on notice

5. **Maintain consent records**
   - Document consent basis
   - Retain for compliance verification

### 3.3 Direct Parental Consent

#### 3.3.1 Obtaining Verifiable Parental Consent

**When:** Collecting children's personal information without school consent (home use, direct sales).

**Steps:**

1. **Provide direct notice to parent**
   - Information to be collected
   - How it will be used
   - Disclosure practices
   - Parent's rights

2. **Select consent method**
   | Method | Use Case | Implementation |
   |--------|----------|----------------|
   | Signed form | Initial enrollment | Send form; receive signed return |
   | Credit card | Paid subscription | Transaction confirms adult |
   | Video conference | Complex situations | Trained staff verifies parent |
   | Government ID | High-risk collection | Check against database |
   | Knowledge-based | Online process | Third-party verification service |
   | Email-plus | Low-risk internal | Email + delayed activation + confirmation |

3. **Execute consent process**
   - Follow selected method
   - Verify parent identity
   - Obtain affirmative consent

4. **Document consent**
   - Consent method used
   - Date of consent
   - Scope of consent (data and uses)
   - Retain proof of consent

5. **Enable service**
   - Activate child's account
   - Begin data collection within consent scope

#### 3.3.2 Renewing Consent

**When:** Material change to data practices or annual verification.

**Steps:**

1. **Identify need for renewal**
   - Material change to collection or use
   - New categories of personal information
   - New third-party disclosures

2. **Notify parent of changes**
   - Describe changes
   - Provide opportunity to consent or decline

3. **Obtain new consent**
   - Use appropriate consent method
   - Document new consent

4. **Handle declined consent**
   - Stop new collection/use
   - May need to terminate service

### 3.4 Notice Requirements

#### 3.4.1 Providing Direct Notice

**When:** Collecting personal information from children (not through school).

**Steps:**

1. **Prepare notice content**
   - Types of information collected
   - How information is used
   - Disclosure practices
   - Parent's rights to review, delete, refuse

2. **Deliver notice**
   - Email to parent's address
   - In-app notice if parent account
   - Clear, prominent display

3. **Confirm receipt**
   - Delivery confirmation
   - Document notice provision

#### 3.4.2 Privacy Policy Requirements

**When:** Operating service used by children.

**Steps:**

1. **Include required elements**
   - Operator name and contact (including email)
   - Types of personal information collected
   - Collection methods (direct and passive)
   - How information is used
   - Third-party disclosures
   - Parent rights and procedures

2. **Post prominently**
   - Link on every page where information collected
   - Accessible from homepage

3. **Keep current**
   - Update when practices change
   - Maintain version history

### 3.5 Data Collection Limits

#### 3.5.1 Reviewing Data Collection

**When:** Designing or modifying product features.

**Steps:**

1. **Identify personal information to collect**
   - Direct identifiers (name, email, phone)
   - Online identifiers (username, persistent identifiers)
   - Photos, videos, audio
   - Geolocation

2. **Evaluate necessity**
   - Is each element necessary for the activity?
   - Can feature work with less information?

3. **Remove unnecessary collection**
   - Collect only what's reasonably necessary
   - Don't condition participation on extra collection

4. **Document collection justification**
   - Business need for each element
   - Retained for compliance review

#### 3.5.2 Prohibiting Behavioral Advertising

**When:** Implementing advertising or tracking features.

**Steps:**

1. **Verify no behavioral advertising**
   - No targeting based on child's information
   - No targeting based on online activity
   - No third-party ad network tracking

2. **If advertising present**
   - Contextual only (based on content, not user)
   - No persistent identifiers to ad networks
   - No data sharing for advertising purposes

3. **Document compliance**
   - Record advertising approach
   - Verify no prohibited practices

### 3.6 Parental Rights

#### 3.6.1 Processing Review Requests

**When:** Parent requests to review child's information.

**Steps:**

1. **Verify parent identity**
   - Government ID
   - Knowledge-based verification
   - Other reasonable method

2. **Compile information**
   - All personal information for the child
   - In readable format

3. **Provide to parent**
   - Secure delivery method
   - Within **[Number, e.g., 30]** days

4. **No charge**
   - Do not charge for access

#### 3.6.2 Processing Deletion Requests

**When:** Parent requests deletion of child's information.

**Steps:**

1. **Verify parent identity**

2. **Delete information**
   - Remove from all systems
   - Within **[Number, e.g., 30]** days

3. **Terminate service if necessary**
   - If information essential, service may end
   - Notify parent

4. **Confirm deletion**
   - Written confirmation to parent

#### 3.6.3 Processing Opt-Out Requests

**When:** Parent refuses further collection.

**Steps:**

1. **Stop further collection**
   - Immediately cease new collection
   - Within **[Number, e.g., 5]** business days

2. **Inform of impact**
   - Explain any service limitations
   - Don't condition on waiving right

3. **Document opt-out**
   - Record in system
   - Ensure future collection blocked

### 3.7 Third-Party Service Providers

#### 3.7.1 Engaging COPPA-Compliant Providers

**When:** Engaging service providers who access children's information.

**Steps:**

1. **Evaluate provider**
   - Privacy and security practices
   - COPPA compliance capability

2. **Include contractual requirements**
   - Use only for specified purpose
   - Security requirements
   - Confidentiality obligations
   - No independent use

3. **Monitor compliance**
   - Periodic assessments
   - Immediate action if non-compliance

4. **Document oversight**
   - Provider agreements
   - Compliance verification

## 4. Escalation

Escalate to Student Data Privacy Officer when:
- Consent validity questioned
- Parent rights dispute
- FTC inquiry or complaint
- Uncertainty about compliance
